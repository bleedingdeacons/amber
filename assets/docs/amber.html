🟠 Amber Plugin

Admin components for intergroup management. Manage service positions, members, meetings, and intergroup meeting attendance — all from your WordPress dashboard.

What Amber Does

Amber adds an Intergroup admin menu to WordPress with four sections: Positions, Members, Groups / Meetings, and Intergroup Meetings. It enriches each of these with custom columns, sortable lists, dashboard widgets, and front-end shortcodes for your public site.

Amber is built for service organisations that need to track who holds which position, when rotations are due, which groups attend intergroup meetings, and how to contact officers — while respecting GDPR through its dependency on the Scrutiny plugin.

Requirements

  • WordPress 6.0 or later
  • PHP 8.1 or later
  • Unity plugin — must be installed and activated (provides the core data layer for positions, members, meetings, and groups)
  • Scrutiny plugin — must be installed and activated (required for GDPR compliance)
  • Advanced Custom Fields (ACF) — used for the custom field editor on all post types
🚨
Amber will not activate without Scrutiny. If Scrutiny is deactivated while Amber is running, Amber will automatically deactivate itself and show a notice explaining why.
⚠️
If Unity is not active, you'll see: "This plugin requires the Unity plugin to be installed and activated." Amber will not load until Unity is running.

Installation

  1. Make sure both the Unity and Scrutiny plugins are installed and activated.
  2. Upload the amber folder to your /wp-content/plugins/ directory, or install via the WordPress plugin uploader.
  3. Go to Plugins → Installed Plugins and activate Amber.
  4. You'll see a new Intergroup menu item in the admin sidebar with four sub-menus: Positions, Members, Groups / Meetings, and Intergroup Meetings.

Positions Overview

Positions represent service roles within your intergroup — such as Chair, Secretary, Treasurer, or GSR Liaison. Each position can be assigned to a member and has properties like a position email, rotation date, minimum sobriety requirement, and term length.

Navigate to Intergroup → Positions to see all positions.

💡
Positions are a custom post type managed through Unity. Amber adds the admin columns, sorting, rotation tracking, and dashboard widget on top.

Positions — Admin List & Columns

The positions admin list includes these custom columns:

ColumnDescription
Position EmailThe generic email address for this role (e.g. chair@example.org). Shown as a clickable mailto link.
Current MemberThe member currently holding the position. Links to their member edit page. Shows "–" if vacant.
StatusColour-coded rotation status (see Rotation Statuses).
Rotation DateThe date the current holder's term ends, in dd/mm/yyyy format. Shows "Not set" if missing.

All columns are sortable — click any column header to sort the list.

Rotation Statuses

Each position displays a colour-coded rotation status in both the admin list and the dashboard widget:

StatusMeaning
✓ X months remainingMore than 3 months until rotation. All is well.
⚑ Due in X months3 months or less until rotation. Time to start finding a replacement.
⏱ Due NowThe rotation month has arrived. The position needs to be handed over.
✗ Overdue by X monthsThe rotation date has passed. Immediate action needed.
⚠ Vacant PositionNo member assigned. The position is empty.
? No Rotation DateA member is assigned but no rotation date has been set.
💡
On the public site, positions with 6 months or fewer until rotation (or vacant / missing rotation date) are highlighted to draw attention.

Members Overview

Members represent individuals serving in your intergroup. Each member record can hold an anonymous name (for privacy), a linked homegroup, a service position, and a flag indicating whether they serve as a GSR (General Service Representative).

Navigate to Intergroup → Members to see all members.

💡
When you save a member, Amber automatically syncs the WordPress post title with the member's Anonymous Name field, so the admin list always shows the correct display name.

Members — Admin List & Columns

The members admin list includes these custom columns:

ColumnDescription
Anonymous NameThe member's display name (e.g. "John D.").
Service PositionThe intergroup position held by this member. Links to the position edit page.
Is GSR?Whether the member is a General Service Representative. Shows ✓ Yes or ✗ No.
HomegroupThe member's home group name.

All columns are sortable.

Groups / Meetings

Navigate to Intergroup → Groups / Meetings to see all meetings. This view uses the TSML (12 Step Meeting List) post type and adds a Group column showing which group each meeting belongs to.

The Group column is sortable, and the search bar is extended so that searching for a group name will also return its meetings.

Meetings Dashboard Widget

A Groups & Meetings widget appears on the WordPress Dashboard, showing a full weekly schedule. Meetings are sorted by day of the week (respecting your WordPress "Week Starts On" setting) and then by start time.

Each row shows:

  • Day — the day name (shown once per day as a header)
  • Group — the parent group (links to the group editor)
  • Meeting — the meeting name (links to the meeting editor), with an "Online" badge for virtual meetings
  • Time — start and end time
  • Location — venue name and address, or an online meeting link
  • Contacts — up to 3 contacts (from the group or meeting), showing name and phone number

Intergroup Meetings

Navigate to Intergroup → Intergroup Meetings to manage records of intergroup assembly meetings. Each intergroup meeting record captures the date, which groups sent representatives, and which officers attended.

Admin List Columns

ColumnDescription
Meeting DateThe date of the intergroup meeting, shown in a human-readable format.
Group AttendeesA list of groups whose GSRs attended (linked to their edit pages).
Officers AttendingA list of officers who attended, showing their anonymous name and position title.
Total AttendeesThe combined count of groups and officers present.

The Meeting Date and Total Attendees columns are sortable.

💡
When selecting officers in the attendance field, Amber shows each member's position name in brackets for easy identification. When selecting groups, the GSR name is appended.

Attendance Tracking

When you save an intergroup meeting, Amber automatically creates and maintains attendance records behind the scenes. This means:

  • Adding a group to the attendees list creates a group attendance record (with the group name and GSR name captured at that point in time)
  • Adding an officer creates an officer attendance record (with their position name captured)
  • Removing a group or officer deletes the corresponding attendance record

These records are used by the Intergroup Meeting dashboards to show attendance history and trends over time.

Shortcodes

Directory List

[directory_list]

Renders a table of all service positions showing the position description, the current holder's public display name, the position email, rotation status, and a link to the position's "About" page.

Position Header

[position_header]

For use on individual position pages. Displays the position title, minimum sobriety requirement, term length, and an "Email Officer" link (if the position is filled).

Position State / Highlight

[position_state]
[position_highlight]

For use on individual position pages. Shows a prominent status message — "Vacant!", "Rotation Overdue!", "Due in X Months", etc. — along with an email link to the current officer.

Position Summary

[position_summary]

For use on individual position pages. Renders the rich-text summary field of the current position.

Dashboard Widgets

Amber adds two widgets to the WordPress admin Dashboard:

Positions & Members

A table of all positions showing the position title, current holder, position email, rotation date, and colour-coded status. Vacant positions and approaching rotations are highlighted. Links go directly to edit pages.

Groups & Meetings

A full weekly meeting schedule (see Meetings Dashboard Widget above for details).

GDPR & Scrutiny

Amber requires the Scrutiny plugin for GDPR compliance. Scrutiny handles the data protection aspects of storing member information (names, contact details, etc.).

🚨
If Scrutiny is deactivated or removed, Amber will automatically deactivate itself to prevent non-compliant handling of personal data. You'll see an admin notice explaining what happened. Reactivate Scrutiny first, then reactivate Amber.

Amber uses anonymous names rather than full names for public-facing displays, and contact information (private emails, phone numbers) is only visible in the admin area.

Troubleshooting

"Scrutiny plugin is required" error on activation

Amber cannot be activated without Scrutiny. Go to Plugins → Installed Plugins, find Scrutiny, and activate it first. Then try activating Amber again.

Amber deactivated itself

This happens when Scrutiny is deactivated while Amber is running. Amber enforces GDPR compliance by refusing to operate without Scrutiny. Reactivate Scrutiny, then reactivate Amber.

Positions show "Not set" for rotation dates

Open the position, assign a member, and set a rotation date using the ACF date picker. Save the position — the status will update automatically.

Custom columns not appearing

Click Screen Options at the top right of any admin list page and ensure the relevant columns are checked. Amber's columns should be visible by default, but Screen Options settings are stored per-user.

Meeting search not finding results by group name

The extended search only works on the Groups / Meetings admin list page. Make sure you're searching from Intergroup → Groups / Meetings, not from the main WordPress search.

FAQ

What's the difference between "Meetings" and "Intergroup Meetings"?

Meetings (under Groups / Meetings) are regular recurring meetings — the weekly schedule of groups. Intergroup Meetings are specific assembly dates where group representatives and officers come together. Think of it as "regular group meetings" vs "the intergroup's own business meetings".

How does the rotation warning threshold work?

Amber highlights positions where rotation is due within 6 months. On the public site (via shortcodes), positions within this threshold show a warning. On the admin side, the status column gives a more granular breakdown: "Due in X months", "Due Now", or "Overdue by X months".

Who can access the Intergroup admin area?

Any user with the edit_posts capability (typically Contributors and above) can view the Intergroup menu and its sub-pages.

Are member names visible to the public?

Only the anonymous name and the position email are displayed on the public site through shortcodes. Private email addresses and phone numbers are only visible in the WordPress admin.

Can I use Amber without the meetings features?

Yes. The Positions and Members sections work independently. If you don't use TSML or the meetings system, those admin pages will simply be empty — nothing will break.
